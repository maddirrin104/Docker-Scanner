D12 - Docker Scanner
A Tool for Scanning Docker Images

Team Members: Vo Minh An, Nguyen Gia Bao, Bui Nguyen Cong Hieu, Sit Khai Dong
1

Abstract

  • Security scanning of Docker images is a crucial step to ensure application safety during deployment. However, existing vulnerability scanning tools are often complex and difficult to integrate into DevOps pipelines.
  • The project aims to build a CLI tool in Python, acting as a wrapper for Trivy, to scan and aggregate vulnerabilities (CVEs) on Docker images with various visual report formats.
  • The tool simplifies the testing process, increases transparency, and helps detect and address vulnerabilities early in the software development lifecycle.
2

Introduction

  • Architecture: Uses Trivy as the core engine, with a Python wrapper leveraging existing libraries (Docker API, SQLite, JSON/YAML parsing, logging, etc.) to perform scans and generate reports.
  • Features: Analyzes Docker images (local/remote), lists installed packages, cross-checks CVE databases (e.g., NVD), and exports reports in various formats (JSON, HTML, CLI, etc.).
  • Data & UI: Utilizes Trivy’s integrated vulnerability database and the Python wrapper for fast and streamlined data handling and report generation.
3

Methodology

  • Use Trivy to scan Docker images for vulnerabilities based on the latest CVE database.
  • Build a Python CLI wrapper that automatically calls Trivy, processes JSON output, and displays results in a tabular format.
  • Workflow: Input image → Wrapper scans → Analyze results → Export visual report.
  • Evaluate effectiveness through vulnerability detection accuracy, automation capability, and user-friendliness.
4

Flow Diagram

Processing Flow: User enters a command via the CLI Wrapper (scanner.py) → The Core Module invokes Trivy for scanning → The results are analyzed and passed to the Report Module to generate output in multiple formats (.json, .html, .csv) → Saved in the scan-results/ directory.

6

Conclusion

  • Achievements: Built a Docker image scanning tool with core features: analyzing vulnerable packages, generating reports with key information in multiple format (terminal, HTML, JSON, CSV), and filtering vulnerabilities by severity with clear, color-coded visualization.
  • Current Limitations: The tool has not yet implemented scan history storage, lacks capabilities to track image evolution over time, and currently relies mainly on the command-line interface, which can be less friendly for new users.
  • Future Development: Enhance history storage and comparison features (between different image versions or scan sessions), enabling security tracking of images over time. Develop a user-friendly GUI (potentially a web-based interface) so users can simply input an image and view scan results directly in their browser without using CLI commands.
